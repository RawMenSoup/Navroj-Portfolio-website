Secure Authentication Demo (Flask)

A comprehensive Flask-based authentication system designed as a penetration testing and secure web development exercise. Built to demonstrate industry best practices in user authentication, session handling, and defensive security controls.

▸ Database: SQLite
▸ ORM: SQLAlchemy

▸ Frontend: HTML + CSS

Project Overview

This project is a Flask-based authentication system built as part of a penetration testing / secure web development exercise. It implements user registration, login, session handling, role-based access, and defensive security controls.

Important Note: This is not production-ready, but it intentionally demonstrates common security patterns and mitigations in a small, readable codebase. The core logic lives in a single Flask app file for clarity and educational value.

  • • Penetration testing practice
  • • Understanding Flask authentication internals

Core Features

Authentication & Authorization

  • ✓ User registration with validated input
  • ✓ Login with session management
  • ✓ Logout with session cleanup
  • ✓ Role-based access (user / admin)
  • ✓ Admin-only access enforcement

Security Controls

  • ✓ Password strength validation (upper/lowercase, numbers, special chars)
  • ✓ Password hashing using PBKDF2 (600k iterations)
  • ✓ Account lockout after failed login attempts
  • ✓ IP-based rate limiting on login

Advanced Security

  • ✓ CSRF protection via Flask-WTF
  • ✓ Secure session cookies (HTTPOnly, SameSite)
  • ✓ Session fixation protection (regeneration)
  • ✓ Login event auditing (IP, user agent, success/failure)

Dashboard & Monitoring

  • ✓ Login-only dashboard access
  • ✓ Recent login activity display
  • ✓ User metadata overview
  • ✓ Security headers (CSP, XSS protection, clickjacking prevention)

Database Schema

users Table

  • id - Primary Key
  • email - Unique Email Address
  • password_hash - PBKDF2 Hash
  • role - 'user' or 'admin'
  • created_at - Registration Timestamp
  • failed_login_attempts - Failed Attempt Counter
  • locked_until - Account Lockout Time
  • last_login - Last Login Timestamp
  • password_changed_at - Password Update Time

login_events Table

  • id - Primary Key
  • email_attempted - Attempted Email
  • ip_address - Source IP
  • user_agent - Browser/Client Info
  • success - Boolean (Success/Failure)
  • failure_reason - Error Details
  • timestamp - Event Time

Important Security Notes

Not for Production

This project is designed for learning and testing purposes only. It is not intended for production deployment in any environment handling real user data or sensitive information.

Configuration Details

  • SECRET_KEY: Falls back to a dev value if not set via environment variable
  • HTTPS: Not enforced; cookies are not Secure by default
  • Database: SQLite is used for simplicity, not scale
  • Rate Limiting: Basic, application-level implementation only
  • CSP: Intentionally permissive to avoid breaking templates

Getting Started

Installation & Setup

1 Clone or Download the Repository 

git clone https://github.com/Navgoat/pen-project.git
cd pen-project

2 Create a Python Virtual Environment 

python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

3 Install Dependencies 

pip install -r requirements.txt

4 Run the Application 

python app.py
# Application will start at http://localhost:5000
